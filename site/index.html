Good questions

How does Agency know what my agents are doing?

It reads your local agent data, read-only. For Claude Code it watches the per-session heartbeat files under ~/.claude/sessions and the transcripts under ~/.claude/projects; for Codex it reads ~/.codex; for opencode, its local database. It cross-references running process IDs to tell which sessions are live and whether each is working, running a shell command, or idle. It never executes, edits, or uploads anything.

Does any of my code or prompts leave my machine?

No. Everything runs locally at localhost:4313. The only thing that can leave is what you explicitly opt into on the leaderboard: a display name and one number (your standardized engineer-years). Never code, prompts, or transcripts.

Which agents are supported?

Claude Code, Codex, and opencode. Auto-discovered, zero config. If a tool isn't installed, Agency simply shows no workers for it.

Do I need to configure anything?

No. One npx command, no build, no dependencies, no API keys. It finds your agents' local data automatically.

Is it free and open source?

Yes. MIT-licensed and on GitHub.
